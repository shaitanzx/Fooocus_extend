Dynamic LoRA – LoRA on steroids.

Standard LoRA is like turning on the lights in a room with a single button. The extended format is a smart home with adjustable lighting levels in each zone. Want it dim in the bedroom and bright in the kitchen? Now you can.

Why do you even need this? (I'll explain it in simple terms)

The problem with standard LoRA

Imagine: you downloaded a cool anime-style LoRA. You apply it with a weight of 0.7 and...
- The picture looks anime-like
- But the faces are blurry, there are unnecessary fingers, the composition is broken.

Why is that? Because the standard <lora:anime:0.7> tag tells the model: "Apply this style EVERYWHERE and TO EVERYTHING equally." It's as if you were painting the entire house with one brush – the walls, the floor, and the ceiling.

Now there's a solution: dynamic control.
Now imagine you could say:
"Hey, model! Take this LoRA, but:
- For large shapes (pose, composition), use 30%
- For small details (textures, style), use 150%
- And only enable this after step 10, when the base is ready."

That's dynamic control.

How does a neural network work?

To understand where to apply weights, you need to know what the "brain" of the generation consists of.

Think of the process of creating an image as building a house:

Stage 1: Foundation and frame (IN blocks)
Stage 2: Finish and style (MID block)
Stage 3: Details and polish (OUT blocks)

Group of blocks Analogy What it's responsible for If you overdo it
IN00–IN08 (Input) Architect, drawing Pose, composition, arrangement of objects Crooked poses, extra limbs, mess in the frame
MID00 (Middle) Interior designer General style, atmosphere, coherence The style will "skew" the entire picture to one side
OUT00–OUT08 (Output) decorator, artist Textures, colors, skin, hair, details Artifacts, "soap", strange patterns on the skin

A simple rule:
- Want to maintain the pose? → Decrease the weight for 'IN'
- Want more style? → Increase the weight for 'OUT'
- Want to change the "mood"? → Play with 'MID'

General Syntax

<lora:Name:w=weight:unet=weight:te=weight:lbw=values:lbwe=expression:start=step:stop=step>

The key feature is that all parameters are optional. If a parameter is omitted, the weight is set to 1 (for w, te, and unet) or remains unchanged (for all other parameters). All parameters are separated by a colon.

The te parameter only works on permanent LORAs.

A permanent LORA is a model that is applied to all generation steps without any modifications.

Dynamic LORA is a model that changes LORA parameters in a given range of generation steps.

Parameter In simple words Example Why do you need
w How much to apply all of LoRA. w=0.8 Quickly weaken or strengthen the effect
unet Weight for the image (visual). unet=1.2 Enhance the style without touching the prompt
te Weight for understanding the text. te=0.5 If the model follows the prompt too literally
lbw Block management IN=0.5,MID=1.0,OUT=1.5 Changing the weight of each block of the model separately
lbwe Layer management IN04-OUT04=attn2=1.3 Managing block layer weights
start When to enable. Step number to start with. start=10 Let him draw the base first, then add the style
stop When to turn off. The final step of application. stop=25 To prevent the style from drying out the finale

If multiple conditions need to be specified in the lbw and lbwe parameters, they are listed comma-separated within this parameter.

If the start parameter is omitted, the model starts running from the very first step.
If the stop parameter is omitted, the model runs until the end of generation.

Examples

"I want the style to appear only at the end"
<lora:WaterColor:unet=1.0:start=15>
The first 14 steps: pure generation. From the 15th: a watercolor style is added.

"Weak composition, but cool textures"
<lora:FantasyArt:unet=0.3:lbw=IN=0.2,OUT=1.5>
LoRA has almost no effect on the pose (`IN=0.2`), but adds a lot of fantasy details (`OUT=1.5`).

LBW: Block Management

First, let's understand how LORA works. Each LoRa has the following 20 blocks:
BASE, IN00, IN01, IN02, IN03, IN04, IN05, IN06, IN07, IN08, MID00, OUT00, OUT01, OUT02, OUT03, OUT04, OUT05, OUT06, OUT07, OUT08.
Since the BASE block basically duplicates the MID00 block, it can be ignored.
Since different LoRas can be trained for different things (style, lighting, character, etc.), they may have empty blocks, and therefore, changing them will not result in any changes.

Method 1: Simple (group)
Set the weight for the entire group of blocks at once:

lbw=IN=0.5,MID=1.0,OUT=1.5

Group What does it mean? When to use
IN=0.5 Input blocks ×0.5 LoRA breaks the composition → weaken
MID=1.0 Middle block ×1.0 We leave the style as is
OUT=1.5 Output block ×1.5 I want more details/textures

You can also specify each block directly (IN00=0.2, IN01=0.5, etc.) or a range of blocks (IN05-OUT05=0.5). In this case, all unspecified blocks will have a weight of 1.0.

Method 2. Named
lbw=IN01=0.3,OUT05=0.9

Group What does it mean? When to useь
IN01=0.3 First input block ×0.5 LoRA breaks the composition → weaken
OUT05=0.9 Fifth output block ×0.9 Reducing detail

Method 3: Exact (Blockwise)
Set the weight for each of the 19 blocks in order:

lbw=0.3,0.4,0.5,0.6,0.7,0.8,0.9,1.0,1.1,1.0,0.9,1.0,1.1,1.2,1.3,1.4,1.5,1.6,1.7

(The first 9 numbers → IN00–IN08, the 10th → MID00, the last 9 → OUT00–OUT08)

If you specify fewer than 19 blocks, only the specified ones are used (starting from the beginning); if you specify more, the extra ones are cut off.

Tip for beginners: Start with the group method (`IN=`, `MID=`, `OUT=`). Exact is for advanced experiments.

LBWE: Surgical Precision (For Geeks and Perfectionists)

The `lbwe` (Elemental) parameter is for when you want to configure not just a "room", but a "specific switch in the room".

### Formula for Success

lbwe=WHERE=WHAT=HOW MUCH

- WHERE: block or range of blocks (`IN05-OUT05`, `MID00`)
- WHAT: layer type (`attn`, `ff`, `to_q`, etc.)
-HOW MUCH: weight multiplier (`1.5`, `0.3`, `-0.5`)

Key Function Visual Impact
to_q Query. How the layer “looks” at itself. Affects the internal structure of objects, posture.
to_k Key. Matching with text. How strictly does the object comply with the prompt?.
to_v Value. Extracting meaning Style of objects, filling with details.
to_out Output. The result of attention. The overall impact of text description on a block.
attn1 Self-Attention. Attention inside the picture. Spatial consistency (so that the hand does not grow to the headве).
attn2 Cross-Attention. Attention to the text. Following the prompt. Enhancement brings the image closer to the text.
ff Feed-Forward (Neural network inside the block). Textures, noise, complex patterns. A common source of artifacts.
proj_in Input data projection. Preparatory stage before processing.
proj_out Output projection of data Finalizing block processing.
in_layers Input convolutions (ResNet). Large forms, primary characteristics.
emb_layers Time/Condition Embedding Layers. Dynamics of changes during generation.
out_layers Output convolutions (ResNet). Detailing, clear boundaries.
skip Skipping processing (Residual). Preservation of original information (without LoRA).

Examples from real life

I want the model to listen to the prompt better, but only in the middle of generation
lbwe=IN04-OUT04=attn2=1.3
Strengthens “attention to the text” only in blocks from the 4th input to the 4th output.

Remove weird texture on skin without losing style
lbwe=OUT00-OUT08=ff=0.7
Reduces the "texture engine" only on output layers.

Make faces clearer, but leave the background alone
lbwe=IN02-OUT02=to_v=1.4
Enhances the detail of objects only in the middle blocks (where faces usually “live”).

Ready-made recipes (Copy and try)

Recipe 1: "Safe Style"
Problem: Lora is cool, but breaks the composition.
Solution: <lora:MyStyle:w=1.0:lbw=IN=0.3,MID=0.8,OUT=1.2>
Weak influence on pose, medium on style, strong on detail.

Recipe 2: “Emergence of style step by step”
Problem: I want the style to “appear” gradually.
Solution: <lora:OilPaint:w=1.0:start=5:stop=25>
The style turns on at the 5th step and turns off at the 25th.

Recipe 3: "Fixing Artifacts"
Problem: Hands/faces have weird texture.
Solution:<lora:FixHands:lbw=OUT:1.0:lbwe=OUT05-OUT08=ff=0.6>
It weakens the "texture engine" only in the final blocks where details are drawn.

Recipe 4: "Two styles in one"
Problem: You want to mix two styles, but in different ways.
Solution: <lora:StyleA:lbw=IN:1.0,OUT:0.5:start=0:stop=10> <lora:StyleB:lbw=IN:0.5,OUT:1.0:start=10:stop=20>
First 10 steps: Style A sets the composition. Next 10: Style B adds detail.

Where to start? (Beginner's plan)

1. Keep it simple. Start with the usual `<lora:name:0.7>`. Works? Great.
2. Try `lbw`. If LoRA breaks the pose → add `lbw=IN=0.5`.
3. Add timing. If the style interferes at the start → `start=10`.
4. Experiment with `lbwe`. Only when you master the previous steps. Start with `attn2` (text attention) or `ff` (texture).

Frequently Asked Questions

Q: This is complicated; I'll get confused by the parameters.
A: Start with one. I added lbw=IN=0.5 → looked at the result. Like it? Add another parameter. Take your time.

Q: What if I specify the wrong key in lbwe?
A: It's okay. If the LoRA file doesn't have such a layer, the rule will simply be ignored. There will be no error.

Q: Will this slow down generation?
A: No! All calculations are done once upon loading. Generation speed does not change.

Q: Where can I find the "correct" values?
A: Trial and error. Start with 0.5 (weak) or 1.5 (strengthen). Look at the result. Remember.

One-Screen Cheat Sheet

BASIC:
w=0.7 # Overall Strength
unet=1.2 # Strength for Image
te=0.5 # Strength for Text Understanding
start=10 # Enable after step 10
stop=25 # Disable after step 25

BLOCKS (lbw):
lbw=IN=0.5 # Reduce Impact on Composition
lbw=MID=1.0 # Leave Style As Is
lbw=OUT:1.5 # Enhance Details

LAYERS (lbwe):
lbwe=ALL=attn2=1.2 # Improve Prompt Listening
lbwe=OUT00-OUT08=ff=0.7 # Remove Artifacts in Details
lbwe=IN02-OUT02=to_v=1.4 # Sharpen Faces in Middle Blocks

READY:
# Safe Style:
<lora:Style:lbw=IN=0.3,MID=0.8,OUT=1.2>

# Style appears later:
<lora:Style:start=10>

# Artifact fixes:
<lora:Fix:lbwe=OUT05-OUT08=ff=0.6>

Top tip: Don't try to learn everything at once. Take one image, one tag, and one parameter. Change → look → remember. This way, step by step, you'll become a master of fine-tuning. Good luck with your creativity!
